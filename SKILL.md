# 学习新领域框架

## 核心公式

```
学习新领域 = 终局锚定 × 流程串联 × 单点穿透
```

## 指导原则

当用户想要学习一个新领域时，按以下框架引导：

### 第一步：WHY - 终局锚定

帮助用户回答：
1. 这个领域解决什么核心问题？
2. 最终交付物/产出是什么？
3. 谁是评判者/用户？
4. 学完能做什么？

### 第二步：WHAT - 流程串联

帮助用户梳理：
1. 从起点到终点有哪些阶段？
2. 每个阶段的输入→加工→输出是什么？
3. 画出完整流程图

```
起点 → 阶段1 → 阶段2 → ... → 终点(交付物)
       [输入|加工|输出]
```

### 第三步：关键概念梳理
帮助用户梳理
1. 每个阶段的输入→加工→输出中，哪些是关键概念？
2. 每个关键概念的定义是什么？
3. 每个关键概念的用途是什么？

### 第四步：HOW - 单点穿透

指导用户：
1. 选择一个最小单元/具体案例
2. 追踪它走完整个流程
3. 通过一个点打通整条线

### 第五步：锚定迁移

帮助用户：
1. 为每个核心概念找一个熟悉的类比
2. 用一句大白话定义每个术语
3. 建立新旧知识的连接

## 输出格式

所有的流程图都要使用线框图，并且要使用清晰的标题。
当用户请求学习新领域时，按以下结构输出：

```markdown
# [领域名称] 学习框架

## 1. 终极目标
根据用户要学习的内容在当前目录下创建一个空的文件夹。
然后在该文件夹下生成 README.md 和 tutorials/ 目录

在 README.md 中写入：
- **解决的问题**：
- **最终交付物**：
- **评判标准**：

## 2. 流程全景图
[流程图]
在 README.md 画出完整详细的流程图

## 3. 核心概念和关键流程解释
根据上面的流程图，解释每个流程的关键点和核心概念，对每个关键概念进行解释，
并在 tutorials/ 目录下创建对应的解释文件，解释文件的目标受众是行业的小白，务必做到通俗易懂，生动有趣。
并在 给出核心概念列表，然后引用对应的解释文件，引导用户打开对应的解释文件去学习相应概念。


## 4. 单点穿透案例
在 tutorials/ 目录下创建一个案例文件，用一个案例来演练整个流程，
案例文件的目标受众是行业的小白，务必做到通俗易懂，生动有趣。

## 5. 学习闭环验证
在 每个核心概念的解释文件中，末尾添加自检清单：
- [ ] 费曼检验：我能向别人讲清楚这个流程吗？
- [ ] 迁移检验：如果换一个类似案例，我能独立完成吗？
- [ ] 深度检验：我能说出每一步的"为什么"吗？
以及每个核心概念的五个常见问题以及相应的解释。

## 一句话口诀

> **终局定方向，流程搭骨架，单点打通关，类比挂认知。**
