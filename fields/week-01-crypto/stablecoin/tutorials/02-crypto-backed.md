# 加密货币抵押型稳定币

## 一句话大白话

**用你的 ETH 当"典当物"，借出稳定币来花；想要回 ETH，就把稳定币还回去。**

这就像去典当行：你把金项链（ETH）押给典当行（智能合约），典当行给你现金（DAI）。区别是这个"典当行"是一段代码，运行在区块链上，没有人类老板，规则完全透明。

---

## 它解决什么问题

### 核心痛点

法币抵押型稳定币有个致命问题：**中心化**。

- Tether 说他有 1000 亿美元储备，你信吗？
- 政府说冻结这个地址，Tether 就得照办
- 万一 Tether 跑路呢？

加密货币抵押型稳定币的回答是：**把抵押品放在链上，用代码自动管理，任何人都可以验证。**

### 使用场景

1. **杠杆做多 ETH**
   ```
   场景：你有 10 ETH，看好后市想加仓

   操作：
   - 抵押 10 ETH
   - 借出 $10,000 DAI
   - 用 DAI 买更多 ETH
   - ETH 涨了 → 卖掉新买的 ETH → 还 DAI → 取回原来的 10 ETH → 赚差价

   效果：不用卖掉 ETH 也能获得流动性
   ```

2. **去中心化信仰者**
   - 不信任任何中心化机构
   - 想要真正"无需许可"的稳定币
   - 不担心被冻结

3. **DeFi 深度玩家**
   - DAI 是很多 DeFi 协议的原生资产
   - 和以太坊生态深度集成
   - 支持更复杂的金融操作

### 类比理解

| 现实世界 | 加密货币抵押型稳定币 |
|---------|---------------------|
| 典当行 | 智能合约（如 MakerDAO） |
| 金项链/手表 | ETH、WBTC 等加密资产 |
| 典当借到的现金 | DAI |
| 典当行老板 | 没有！代码自动执行 |
| 赎回条件 | 还款 + 利息（稳定费） |
| 违约处理 | 自动清算（被强制平仓） |

## 什么时候用 / 什么时候别用

### ✅ 适合使用的场景

| 场景 | 说明 |
|------|------|
| 追求去中心化 | 没有中心化机构可以冻结你的资金 |
| 想要杠杆做多 | 抵押 ETH 借 DAI，买更多 ETH |
| 需要透明度 | 链上可验证，抵押率实时可查 |
| 长期持有者 | 不想卖币但需要流动性 |
| 参与 DeFi 治理 | 可以参与 MakerDAO 等协议的治理 |

### ❌ 不适合使用的场景

| 场景 | 原因 |
|------|------|
| 追求资本效率 | 超额抵押意味着资金利用率低（抵押 150 才能借 100） |
| 害怕清算 | 币价暴跌可能被强制清算，损失更大 |
| 新手入门 | 机制复杂，操作门槛高 |
| 小额使用 | Gas 费相对固定，小额操作不划算 |
| 极端行情 | 暴跌时可能出现清算拥堵，损失加大 |

## 它不是什么

### 常见混淆点

| 误解 | 真相 |
|------|------|
| "DAI 和 USDT 一样" | DAI 是超额抵押生成的，USDT 是 1:1 法币储备 |
| "抵押后 ETH 就没了" | ETH 只是被锁住，还款后可以取回 |
| "去中心化 = 无风险" | 智能合约也有漏洞风险，清算机制也会失灵 |
| "抵押率是固定的" | 抵押率会随币价变化，需要持续关注 |
| "DAI 永远等于 1 美元" | DAI 也会脱锚，只是有机制会将其拉回 |

### Anti-Patterns（反模式）

1. **抵押率贴着清算线**
   - 问题：稍有波动就被清算
   - 建议：保持抵押率在安全线以上（如 200% 以上）

2. **不设置价格预警**
   - 问题：睡一觉起来发现被清算了
   - 建议：使用 DeFi Saver 等工具自动补仓或设置预警

3. **在波动剧烈时操作**
   - 问题：Gas 费飙升，交易延迟，可能在最坏时机被清算
   - 建议：避开市场剧烈波动期进行操作

## 新手最常踩的 3 个坑

### 坑 1：不理解"超额抵押"，以为能 1:1 借

**错误想法**：
> "我有价值 $10,000 的 ETH，应该能借 $10,000 DAI 吧？"

**真相**：
```
超额抵押的含义：
─────────────────────────────────────────────

抵押品价值必须 > 借款金额 × 最低抵押率

例如，150% 抵押率意味着：
- 抵押 $150 → 最多借 $100
- 抵押 $10,000 → 最多借 $6,666

为什么要超额？
因为抵押品（ETH）价格会波动！
$10,000 的 ETH 可能明天就变成 $8,000
超额部分就是"安全垫"

实际建议：
不要借到极限，保持 200%+ 的抵押率
给价格波动留出空间
```

### 坑 2：忘记稳定费，以为借款免费

**错误想法**：
> "我借了 DAI，用完再还就行了，不用额外付钱吧？"

**真相**：
```
稳定费（Stability Fee）详解：
─────────────────────────────────────────────

稳定费 = 借款利息，持续累积

例如：
- 借款：10,000 DAI
- 稳定费率：2% 年化
- 借款时间：1 年

到期需要还：10,000 + 200 = 10,200 DAI

注意事项：
1. 稳定费按复利计算，每秒累积
2. 费率由治理投票决定，可能变化
3. 不同抵押品类型费率不同（ETH-A、ETH-B、ETH-C）
4. 还款时必须先还利息，再还本金

查看当前费率：
https://daistats.com/
```

### 坑 3：清算时 Gas 费飙升，补仓不及时

**场景**：
```
市场暴跌时的噩梦：
─────────────────────────────────────────────

时间线：
09:00  ETH = $2,000  抵押率 200%  🟢
09:30  ETH = $1,800  抵押率 180%  🟢
10:00  ETH = $1,600  抵押率 160%  🟡 你收到预警
10:05  你想补仓，提交交易
10:06  Gas 费从 30 Gwei 飙升到 500 Gwei
10:07  你的交易卡在队列里
10:10  ETH = $1,500  抵押率 150%  🔴 被清算！
10:11  你的补仓交易终于确认...但已经太晚了

损失：
- 清算罚金 13%
- 被清算时价格可能不利
- 本来可以避免的损失
```

**解决方案**：
```
预防措施：
─────────────────────────────────────────────

1. 使用自动化工具
   - DeFi Saver：自动补仓/自动减仓
   - 设置触发条件，无需手动操作

2. 保守的抵押率
   - 不要贪心借太多
   - 抵押率 250%+ 更安全

3. 设置多重预警
   - 使用 DeBank、Zerion 等工具
   - 设置 Telegram/Discord 机器人提醒

4. 准备应急资金
   - 钱包里留一些 ETH 和 DAI
   - 紧急时刻可以快速补仓
```

## 自测题

### 问题 1：计算题
你抵押了 5 ETH（当前价格 $2,000），借出了 5,000 DAI。
- 你的当前抵押率是多少？
- 清算价格是多少（假设清算比率 150%）？
- ETH 跌到 $1,800 时，你的抵押率变成多少？

<details>
<summary>查看答案</summary>

**当前抵押率**：
- 抵押品价值 = 5 × $2,000 = $10,000
- 抵押率 = $10,000 ÷ $5,000 = 200%

**清算价格**：
- 清算价格 = 债务 × 清算比率 ÷ 抵押品数量
- 清算价格 = $5,000 × 1.5 ÷ 5 = $1,500
- 即 ETH 跌到 $1,500 时会被清算

**ETH = $1,800 时的抵押率**：
- 抵押品价值 = 5 × $1,800 = $9,000
- 抵押率 = $9,000 ÷ $5,000 = 180%
- 状态：🟡 警告区域，但还未触发清算
</details>

### 问题 2：策略题
市场剧烈波动，ETH 从 $2,000 快速下跌。你有一个 Vault，当前抵押率 180%，清算价格 $1,500。你应该怎么办？

<details>
<summary>查看答案</summary>

**选项分析**：

1. **补充抵押品**
   - 存入更多 ETH，提高抵押率
   - 优点：保住仓位，不用卖币
   - 缺点：需要额外资金，如果继续下跌损失更大

2. **偿还部分债务**
   - 用手里的 DAI 还一部分
   - 优点：降低债务，提高抵押率
   - 缺点：需要有 DAI 或买入 DAI

3. **部分清仓**
   - 取出部分抵押品，还掉对应债务
   - 优点：主动降低风险敞口
   - 缺点：可能错过反弹

4. **使用 DeFi Saver 自动化**
   - 设置自动补仓/减仓
   - 优点：无需盯盘
   - 缺点：需要支付服务费

**建议策略**：
- 如果看好后市：补充抵押品
- 如果不确定：部分清仓，落袋为安
- 无论如何：设置自动化工具，防止睡觉时被清算
</details>

### 问题 3：对比题
为什么说 DAI 比 USDT "更去中心化"？这带来了什么优缺点？

<details>
<summary>查看答案</summary>

**DAI 更去中心化的原因**：

| 维度 | USDT | DAI |
|------|------|-----|
| 发行方 | Tether 公司 | 智能合约（无人类控制） |
| 抵押品存放 | 银行账户 | 链上智能合约 |
| 可验证性 | 依赖审计报告 | 链上实时可查 |
| 冻结能力 | 可以冻结地址 | 无法冻结（除非合约升级） |
| 赎回机制 | 找 Tether 赎回 | 和合约交互即可 |

**去中心化的优点**：
1. 无需信任第三方
2. 抗审查，不能被冻结
3. 透明可验证
4. 7×24 小时运作

**去中心化的缺点**：
1. 资本效率低（需要超额抵押）
2. 智能合约风险（代码漏洞）
3. 极端行情时清算机制可能失灵
4. 用户门槛高，操作复杂
5. 依赖预言机，存在喂价风险

**结论**：去中心化是一种取舍，不是单纯的"更好"或"更差"。
</details>
