# 算法稳定币

## 一句话大白话

**没有抵押品，全靠"印钞"和"烧钱"来维持价格稳定——像央行一样，但由代码自动执行。**

想象一下：如果美联储不是一群人开会决定印多少钱，而是一段程序自动根据通胀/通缩数据调节货币供应量。这就是算法稳定币的理念——用代码取代央行。

---

## 它解决什么问题

### 核心痛点

1. **法币抵押型**：需要信任中心化机构，效率受限于银行系统
2. **加密货币抵押型**：需要超额抵押，资本效率低

算法稳定币的野心是：**既不需要信任任何人，也不需要锁住大量资金，纯靠数学和博弈论维持稳定。**

### 使用场景（理想情况）

1. **极致的资本效率**
   - 理论上可以 1:1 甚至无抵押发行
   - 不用锁住大量资金

2. **完全去中心化**
   - 没有中心化发行方
   - 没有可被冻结的储备金

3. **可扩展性**
   - 需要多少就印多少
   - 不受抵押品限制

### 类比理解

| 现实世界 | 算法稳定币 |
|---------|-----------|
| 央行 | 智能合约算法 |
| 印钞票 | 增发代币 |
| 回收钞票 | 销毁代币 |
| 利率调控 | 套利激励 |
| 通胀/通缩 | 价格高于/低于锚定值 |

---

## 什么时候用 / 什么时候别用

### ⚠️ 重要警告

**绝大多数算法稳定币已经失败或正在挣扎。**

2022 年 5 月 UST/LUNA 的崩盘是加密史上最大的灾难之一，导致超过 400 亿美元归零。在学习算法稳定币时，请把它当作**案例研究**而非投资建议。

### ✅ 适合了解的场景

| 场景 | 说明 |
|------|------|
| 学术研究 | 理解货币政策和博弈论 |
| 机制设计 | 学习激励机制如何运作 |
| 风险教育 | 理解"死亡螺旋"是怎么发生的 |
| 创新探索 | 关注新一代混合型稳定币 |

### ❌ 应该避免的场景

| 场景 | 原因 |
|------|------|
| 大额资金存储 | 极高的归零风险 |
| 作为主要稳定币使用 | 流动性差，脱锚风险高 |
| 追求"高收益" | 高收益来自高风险，可能血本无归 |
| 不了解机制就参与 | 一定要先搞懂再决定是否参与 |

## 它不是什么

### 常见混淆点

| 误解 | 真相 |
|------|------|
| "算法稳定币更先进" | 更复杂 ≠ 更好，目前成功案例很少 |
| "代码控制所以更安全" | 代码可能有漏洞，机制可能有缺陷 |
| "没有抵押品也能稳定" | 大多数尝试都失败了，需要某种形式的价值支撑 |
| "高收益是可持续的" | 高收益往往来自新资金流入，类似庞氏结构 |
| "这次不一样" | 每个失败的项目都这么说过 |

### Anti-Patterns（反模式）

1. **被高收益吸引**
   ```
   常见陷阱：
   "存入稳定币，年化收益 20%！"

   真相：
   - 收益来自哪里？
   - 如果是来自代币增发 → 本质是稀释
   - 如果是来自新用户存款 → 庞氏结构
   - 如果是来自真实收益 → 请验证
   ```

2. **不理解机制就重仓**
   ```
   危险行为：
   - "别人都在买，我也买"
   - "涨了这么多，应该没问题"
   - "项目方很有名，应该靠谱"

   正确做法：
   - 理解价格是如何维持的
   - 理解崩盘时会发生什么
   - 只投入能承受归零的金额
   ```

3. **忽视"死亡螺旋"风险**
   ```
   UST 的教训：
   - 看起来很稳定，直到突然不稳定
   - 脱锚一旦开始，可能无法逆转
   - 所有"稳定"机制在恐慌时都可能失效
   ```

```
═══════════════════════════════════════════════════════════════

代表项目：Ampleforth (AMPL)

核心思想：
不追求单个代币价格稳定，而是追求你持有的"购买力"稳定

│                                                             │

│                                                             │

│  触发：正向 Rebase（增发）                                  │
│  效果：每个人钱包里的 AMPL 数量增加                          │
│  结果：供应↑ → 价格↓ → 回归 $1                             │
│                                                             │
│  你的钱包变化：                                              │
│  之前：100 AMPL × $1.10 = $110                              │
│  Rebase 后：110 AMPL × $1.00 = $110                         │
│  （数量增加，但总价值不变）                                  │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  情况二：AMPL 价格 < $1（比如 $0.90）                        │
│  ─────────────────────────────────────────                  │
│  触发：负向 Rebase（收缩）                                  │
│  效果：每个人钱包里的 AMPL 数量减少                          │
│  结果：供应↓ → 价格↑ → 回归 $1                             │
│                                                             │
│  你的钱包变化：                                              │
│  之前：100 AMPL × $0.90 = $90                               │
│  Rebase 后：90 AMPL × $1.00 = $90                           │
│  （数量减少，但总价值不变）                                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘

问题：
- 价格波动没有消失，只是转移到了数量上
- 用户心理上难以接受"钱包里的币变少了"
- 与 DeFi 协议集成困难
```

### 例子 2：双代币机制（UST/LUNA 模型）

```
═══════════════════════════════════════════════════════════════
           双代币算法稳定币（以 UST/LUNA 为例）
═══════════════════════════════════════════════════════════════

两个代币：
- UST：稳定币，目标锚定 $1
- LUNA：波动代币，吸收 UST 的波动性

核心机制：1 UST 永远可以兑换价值 $1 的 LUNA

┌─────────────────────────────────────────────────────────────┐
│                     铸造/赎回机制                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  铸造 UST：                                                 │
│  ──────────                                                 │
│  用户销毁 $1 价值的 LUNA → 获得 1 UST                       │
│                                                             │
│  例如：LUNA 价格 = $100                                     │
│  销毁 0.01 LUNA → 铸造 1 UST                                │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  赎回 UST：                                                 │
│  ──────────                                                 │
│  用户销毁 1 UST → 获得 $1 价值的 LUNA                       │
│                                                             │
│  例如：LUNA 价格 = $100                                     │
│  销毁 1 UST → 铸造 0.01 LUNA                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘

套利机制维持锚定：

场景一：UST > $1（比如 $1.02）
─────────────────────────────────────────
套利者操作：
1. 销毁 $1 的 LUNA → 铸造 1 UST
2. 在市场上以 $1.02 卖出 UST
3. 赚取 $0.02 利润
结果：UST 供应↑，价格↓，回归 $1

场景二：UST < $1（比如 $0.98）
─────────────────────────────────────────
套利者操作：
1. 在市场上以 $0.98 买入 1 UST
2. 销毁 1 UST → 铸造 $1 的 LUNA
3. 卖出 LUNA 获得 $1
4. 赚取 $0.02 利润
结果：UST 需求↑（被买入），价格↑，回归 $1
```

## 新手最常踩的 3 个坑

### 坑 1：被"稳定"两个字迷惑

**错误想法**：
> "叫稳定币，应该很稳定吧？年化 20% 还稳定，太香了！"

**真相**：
```
"稳定"的含义：
─────────────────────────────────────────

法币抵押型（如 USDT）：
- 稳定来源：银行账户里的真金白银
- 风险：发行方信誉

加密抵押型（如 DAI）：
- 稳定来源：超额的抵押品
- 风险：抵押品暴跌 + 清算失灵

算法稳定币：
- 稳定来源：数学公式 + 市场博弈
- 风险：信心崩塌 → 一切归零

关键区别：
前两种有"真实资产"支撑
算法稳定币只有"信念"支撑
信念可以在一夜之间消失
```

### 坑 2：不理解"高收益"的来源

**常见套路**：
```
项目方的话术：
"存入我们的稳定币，享受 20% 年化收益！"

你应该问的问题：
─────────────────────────────────────────

Q1: 收益从哪里来？
- 真实收益（借贷利差、交易手续费）？
- 代币增发（稀释你的份额）？
- 新用户存款（庞氏结构）？

Q2: 这个收益率可持续吗？
- 如果依赖新资金流入，迟早会枯竭
- 如果依赖代币增发，你赚的是数字不是价值

Q3: 风险和收益匹配吗？
- 银行存款 2-3%
- 国债 4-5%
- 20%+ 意味着什么级别的风险？

UST/Anchor 的教训：
- 20% 收益来自项目方补贴
- 补贴资金来自 LUNA 的销售
- 当 LUNA 崩盘，一切都没了
```

### 坑 3：低估"死亡螺旋"的速度

**心理陷阱**：
```
常见心态：
─────────────────────────────────────────

"跌了 5%，抄底！"
      │
      ▼
"跌了 20%，加仓摊平成本！"
      │
      ▼
"跌了 50%，应该到底了吧？"
      │
      ▼
"跌了 90%，反正也亏这么多了，不卖了"
      │
      ▼
"归零了......"

UST 崩盘速度：
- 5月7日：$1.00
- 5月9日：$0.60（2天跌40%）
- 5月12日：$0.10（5天跌90%）
- 5月13日：接近$0（6天归零）

教训：
- 算法稳定币一旦开始脱锚，可能无法恢复
- 不要试图"抄底"正在崩盘的算法稳定币
- 逃跑要趁早，不要等着"反弹"
```

## 自测题

### 问题 1：机制理解
在 UST/LUNA 模型中，当 UST 价格跌到 $0.90 时，套利者应该如何操作？这个操作为什么在极端情况下会失效？

<details>
<summary>查看答案</summary>

**理论上的套利操作**：
1. 在市场上以 $0.90 买入 1 UST
2. 将 1 UST 销毁，铸造价值 $1 的 LUNA
3. 卖出 LUNA 获得 $1
4. 利润 = $1 - $0.90 = $0.10

**为什么会失效**：

1. **LUNA 价格也在暴跌**
   - 当你铸造出 LUNA 时，LUNA 可能已经跌了
   - 你卖出 LUNA 可能只能拿到 $0.80

2. **流动性枯竭**
   - 市场上没人愿意买 LUNA
   - 你有 LUNA 也卖不掉

3. **机制本身加速崩盘**
   - 每次赎回都会铸造新 LUNA
   - 新 LUNA 供应增加 → 价格下跌
   - 需要铸造更多 LUNA → 供应继续增加
   - 死亡螺旋形成

4. **信心完全丧失**
   - 没有人相信 UST 会回到 $1
   - 套利者也不愿意冒险
   - 机制停止运作
</details>

### 问题 2：风险评估
一个新的算法稳定币项目承诺 30% 年化收益，你应该问哪些问题来评估风险？

<details>
<summary>查看答案</summary>

**必须搞清楚的问题**：

1. **收益来源**
   - 是真实的业务收入吗？
   - 是代币增发稀释吗？
   - 是新用户存款（庞氏结构）吗？

2. **稳定机制**
   - 用什么方式维持锚定？
   - 有没有抵押品？抵押率多少？
   - 极端情况下会发生什么？

3. **项目背景**
   - 团队是谁？有没有跑路历史？
   - 代码是否开源和审计？
   - 有没有类似项目失败过？

4. **市场数据**
   - 总市值和流动性如何？
   - 有没有大户集中持有？
   - 历史脱锚情况？

5. **退出机制**
   - 能随时赎回吗？有锁仓期吗？
   - 大额赎回会发生什么？
   - 如果项目崩盘，能跑掉吗？

**红旗信号**：
- 收益来源说不清楚 → 危险
- 没有任何抵押品 → 非常危险
- 团队匿名 → 更危险
- 代码没审计 → 最危险
</details>

### 问题 3：比较分析
FRAX 是"混合型"稳定币，它和纯算法稳定币（如 UST）有什么区别？为什么它相对更稳定？

<details>
<summary>查看答案</summary>

**FRAX 的设计**：
- 部分抵押（目前约 90%+ 由 USDC 等抵押）
- 部分算法（FXS 代币吸收波动）
- 抵押率动态调整（根据市场情况）

**与纯算法稳定币的区别**：

| 特性 | UST | FRAX |
|------|-----|------|
| 抵押品 | 无 | 有（USDC 等） |
| 抵押率 | 0% | 90%+（动态） |
| 波动吸收 | 100% 靠 LUNA | 主要靠抵押品，部分靠 FXS |
| 最坏情况 | 完全归零 | 至少能赎回抵押品部分 |

**为什么更稳定**：

1. **有安全垫**
   - 即使 FXS 归零，还有 90%+ 的抵押品
   - 不会像 UST 那样完全归零

2. **动态调整**
   - 市场恐慌时，可以提高抵押率
   - 稳定时，可以降低抵押率提高效率

3. **渐进式去中心化**
   - 不追求 100% 算法
   - 在稳定性和效率之间寻找平衡

4. **吸取前人教训**
   - 经历了多次行业周期
   - 机制更加审慎

**但仍有风险**：
- 依赖 USDC 等中心化稳定币作为抵押
- FXS 代币市值仍需支撑系统
- 复杂机制可能有未知漏洞
</details>

## 算法稳定币发展简史

| 时期 | 代表项目 | 机制 | 结局 |
|------|---------|------|------|
| 2018 | Basis (未上线) | 债券 + 股票 + 稳定币三代币 | 监管压力，项目终止 |
| 2020 | Ampleforth | Rebase（弹性供应） | 存活但波动大 |
| 2020 | Empty Set Dollar | 优惠券 + Rebase | 失败，脱锚 |
| 2020 | Basis Cash | 仿 Basis 三代币 | 失败，脱锚 |
| 2021 | Fei Protocol | PCV（协议控制价值） | 艰难存活，后合并 |
| 2021-2022 | UST/LUNA | 双代币销毁铸造 | 崩盘，损失 400 亿 |
| 2020-至今 | FRAX | 混合型（部分抵押） | 相对稳定 |
| 2022-至今 | crvUSD | 超额抵押 + LLAMMA | 新尝试，观察中 |

**行业共识**：纯算法稳定币很可能是死胡同，未来方向是混合型或创新的抵押机制。
