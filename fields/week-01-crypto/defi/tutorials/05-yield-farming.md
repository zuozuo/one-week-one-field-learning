# 收益耕作（Yield Farming）

## 一句话大白话

**收益耕作就是把你的加密资产「种」到各种 DeFi 协议里，像农民种田一样「收割」代币奖励。APY 可能很高，但风险也很大——天上不会掉馅饼。**

---

## 它解决什么问题

### 闲置资产的困境

```
传统方式：
┌─────────────────────────────────────────────────────────────┐
│  你有 10 ETH 放在钱包里                                      │
│                                                             │
│  什么也不做 → 收益 = 0                                       │
│  （还可能因为通胀而贬值）                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘

DeFi 收益耕作：
┌─────────────────────────────────────────────────────────────┐
│  你的 10 ETH 可以：                                          │
│                                                             │
│  存入借贷协议 → 赚取利息（3-8% APY）                         │
│  存入流动性池 → 赚取手续费 + 代币奖励（10-100%+ APY）         │
│  质押 LP Token → 赚取额外奖励                                │
│  循环嵌套 → 多层收益叠加                                     │
│                                                             │
│  资产不再闲置，而是在「干活赚钱」                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### DeFi 协议为什么要给你奖励？

```
答案：他们需要你的钱！

┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  新协议：「我们刚上线，没人用怎么办？」                       │
│                                                             │
│  解决方案：发代币！                                          │
│                                                             │
│  「只要你来存钱，我们就送你协议代币！」                       │
│                                                             │
│  结果：                                                      │
│  • 用户涌入，协议 TVL（锁仓量）上涨                          │
│  • TVL 高 → 看起来可信 → 更多用户来                          │
│  • 代币有需求 → 价格上涨 → 收益更高                          │
│  • 正向飞轮转起来了                                          │
│                                                             │
│  但是：当奖励减少或代币暴跌时，飞轮反转...                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 收益耕作的层级结构

### 第一层：基础收益

```
┌─────────────────────────────────────────────────────────────┐
│  基础收益 = 协议的「本职工作」产生的收益                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  借贷协议（Aave）                                           │
│  └── 存款利息（来自借款人支付）                              │
│                                                             │
│  DEX（Uniswap）                                             │
│  └── 交易手续费（来自交易者支付）                            │
│                                                             │
│  这是「真实收益」，有实际业务支撑                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 第二层：代币激励

```
┌─────────────────────────────────────────────────────────────┐
│  代币激励 = 协议「印钱」发给你                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Compound 送 COMP 代币                                      │
│  Aave 送 stkAAVE                                           │
│  SushiSwap 送 SUSHI                                        │
│  Curve 送 CRV                                              │
│                                                             │
│  这是「通胀收益」，需要有人接盘                              │
│  如果代币暴跌，你的收益也暴跌                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 什么时候用 / 什么时候别用

### ✅ 适合收益耕作的场景

| 场景 | 原因 |
|------|------|
| 长期持有的资产 | 反正要放着，不如让它产生收益 |
| 理解风险且能承受 | 知道自己在做什么 |
| 有时间和精力监控 | 市场变化快，需要及时调整 |
| 资金量适中 | 太小 Gas 费不划算，太大怕风险 |

### ❌ 不适合的场景

| 场景 | 原因 |
|------|------|
| 追着高 APY 跑 | 高收益往往是新项目，风险最大 |
| 不懂原理就冲 | 不知道钱怎么赚的，就不知道怎么亏的 |
| 借钱去挖矿 | 双重杠杆，血亏 |
| 被 FOMO 裹挟 | 看别人赚钱就冲，结果当接盘侠 |

## 它不是什么（常见混淆点）

### ❌ 高 APY 不是「真的能赚那么多」

```
显示的 APY 陷阱：

┌─────────────────────────────────────────────────────────────┐
│  某池子显示 APY 500%                                        │
│                                                             │
│  实际情况：                                                  │
│  • 500% 是按当前瞬时数据年化                                 │
│  • 奖励代币价格可能暴跌 80%                                  │
│  • 无常损失可能吃掉 20% 本金                                 │
│  • 实际一年后收益可能是负的                                  │
│                                                             │
│  记住：APY 是「如果一切不变」的假设                          │
│        但 DeFi 世界，一切都在变                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### ❌ 收益耕作不是「躺赚」

```
误解：存进去就不用管了

实际：
• 利率随时变化，可能从 20% 跌到 2%
• 奖励代币随时可能暴跌
• 智能合约可能被攻击
• 新的更好的机会随时出现
• 需要频繁关注、调仓

收益耕作是「主动投资」，不是「被动收入」
```

### ❌ TVL 高不是「一定安全」

```
误解：TVL（锁仓量）高的协议就安全

实际：
• TVL 高只说明钱多，不代表没 bug
• 越多钱 → 黑客越有动力攻击
• 资金可以随时撤走，TVL 随时暴跌
• 2022 年 Terra/UST 几百亿 TVL，照样归零
```

## 最小例子：计算真实收益

### 场景设定

```
你准备用 $10,000 做收益耕作
选择 ETH-USDC LP 池
协议显示：
- 交易费 APR：5%
- 代币奖励 APR：45%
- 合计 APY：约 60%（复利）
```

### 真实收益计算

```python
# 初始投入
initial_capital = 10000

# 显示的收益率
trading_fee_apr = 0.05
token_reward_apr = 0.45
displayed_apr = trading_fee_apr + token_reward_apr  # 50%

# 一年后「理想」收益
ideal_profit = initial_capital * displayed_apr  # $5,000

# 但是，考虑真实世界的损耗：

# 1. 无常损失（假设 ETH 涨 50%）
impermanent_loss = 0.02  # 约 2%
il_cost = initial_capital * impermanent_loss  # $200

# 2. 奖励代币贬值（假设跌 50%）
token_reward_nominal = initial_capital * token_reward_apr  # $4,500
token_depreciation = 0.50
actual_token_value = token_reward_nominal * (1 - token_depreciation)  # $2,250

# 3. Gas 费（假设每月调仓一次，每次 $20）
gas_cost = 20 * 12  # $240

# 4. 交易费收益（相对稳定）
trading_fee_profit = initial_capital * trading_fee_apr  # $500

# 真实收益计算
actual_profit = trading_fee_profit + actual_token_value - il_cost - gas_cost
# = 500 + 2250 - 200 - 240 = $2,310

actual_apr = actual_profit / initial_capital
# = 23.1%

print(f"显示 APR: {displayed_apr:.0%}")
print(f"实际收益: ${actual_profit:.0f}")
print(f"实际 APR: {actual_apr:.1%}")
print(f"打折比例: {actual_apr/displayed_apr:.0%}")
```

输出：
```
显示 APR: 50%
实际收益: $2310
实际 APR: 23.1%
打折比例: 46%

结论：实际收益往往只有显示的一半甚至更少
```

## 新手最常踩的 3 个坑

### 坑 1：追高 APY，成为接盘侠

```
场景：
你看到某新项目 APY 10000%
兴冲冲冲进去
三天后代币暴跌 95%
你的本金加收益全没了

这就是「挖-提-卖」循环：
早期玩家：挖到代币 → 立刻卖掉 → 赚钱走人
后来玩家：高价买入 → 代币暴跌 → 接盘亏损

如何避免：
• 不冲新项目（至少等一个月）
• 挖到的代币及时变现
• 计算「回本周期」，太长的别碰
```

### 坑 2：忽视 Gas 费

```
场景：
你用 $500 在以太坊主网挖矿
每次操作 Gas $30-50
授权、存入、领取、卖出...
一通操作下来，Gas 花了 $200

$500 本金，Gas 就占了 40%！

正确做法：
• 小资金去 Layer2 或便宜的链
• 计算 Gas 占比，超过 5% 要考虑
• 减少操作频率，批量处理
• 使用 Gas 追踪工具选低峰期
```

### 坑 3：不做风险隔离

```
场景：
你把所有资产放进一个协议
这个协议被黑了
血本无归

正确做法：
┌─────────────────────────────────────────────────────────────┐
│  风险分散原则：                                              │
│                                                             │
│  • 不要超过 20% 资产放在单一协议                             │
│  • 不要把全部资产放在 DeFi 里                                │
│  • 优先使用经过时间考验的老协议                              │
│  • 关注审计报告和安全事件                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 收益聚合器

### 什么是收益聚合器？

```
问题：
• 手动挖矿太累
• 复利需要频繁操作
• Gas 费高
• 策略复杂

解决方案：收益聚合器（Yield Aggregator）

代表项目：Yearn Finance、Beefy、Convex

原理：
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   你的资产 → 存入聚合器 → 聚合器帮你：                        │
│                            ├── 自动选择最优策略               │
│                            ├── 自动复利                       │
│                            ├── 分摊 Gas 费                    │
│                            └── 自动调仓                       │
│                                                             │
│   收费：通常收取收益的 10-20%                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 要不要用聚合器？

| 情况 | 建议 |
|------|------|
| 资金小（< $5,000） | 用！Gas 费分摊很划算 |
| 没时间管 | 用！自动化省心 |
| 想最大化收益 | 看情况，管理费是成本 |
| 想学习原理 | 不用，自己操作一遍更懂 |

## 自测题

### 基础理解
1. 收益耕作的收益来源有哪些？哪些是「真实收益」，哪些是「通胀收益」？
2. 为什么新项目的 APY 往往特别高？这背后的经济逻辑是什么？
3. 显示的 APY 和实际能拿到的收益为什么会有差距？

### 实践应用
4. 你有 $5,000 想做收益耕作，应该选什么策略？为什么？
5. 一个项目显示 APY 200%，你应该做哪些检查再决定是否参与？

### 深度思考
6. 如果所有挖到的代币都会被立刻卖掉，这个系统能持续吗？
7. 收益耕作和传统理财（比如货币基金）的本质区别是什么？

---

## 答案提示

1. 交易手续费、借贷利息是真实收益（有实际业务支撑）；代币奖励是通胀收益（协议「印钱」发的）
2. 新项目需要用户，高 APY 吸引资金；实际上是用代币「补贴」用户，代币来自团队/社区份额
3. 奖励代币会贬值、有无常损失、有 Gas 成本、利率会下降
4. 低风险策略：稳定币挖矿；原因：资金量不大，不值得冒高风险
5. 检查：团队背景、审计报告、TVL 变化、代币经济学、解锁时间表、社区口碑
6. 不能持续。需要有人买入代币才能维持价格。纯「挖-卖」会让代币归零
7. 传统理财收益来自借贷利差，相对稳定；收益耕作部分来自协议补贴，不可持续；风险完全不同
