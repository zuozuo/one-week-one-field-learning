# 稳定币（Stablecoin）

## 一句话大白话

**稳定币就是加密世界里的「美元」——一个代币永远值 1 美元（或接近），让你在波动剧烈的加密市场有个安全的避风港。**

---

## 它解决什么问题

### 加密货币的波动噩梦

```
场景：你想用 BTC 买杯咖啡

早上 10 点：1 BTC = $60,000
下单咖啡 = 0.0001 BTC = $6

下午 2 点：1 BTC = $55,000
同样的 0.0001 BTC = $5.5

老板：？？？我的咖啡到底该收多少钱？？？

结论：高波动的币种不适合作为日常支付手段
```

### 稳定币的解决方案

```
场景：用稳定币买咖啡

早上 10 点：1 USDC = $1.00
下单咖啡 = 6 USDC = $6

下午 2 点：1 USDC = $1.00
同样的 6 USDC = $6

老板：稳定！

而且：
✅ 24 小时全球转账
✅ 几分钟到账（vs 电汇 3-5 天）
✅ 手续费低廉
✅ 无国境限制
```

## 稳定币的三种类型

### 1. 法币抵押型（最常见）

```
代表：USDT、USDC

工作原理：
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   你 ────▶ 存入 $1 真实美元 ────▶ 发行公司（Circle/Tether）  │
│                                         │                   │
│                                         ▼                   │
│   你 ◀──── 收到 1 USDC/USDT ◀────  铸造代币                 │
│                                                             │
│   理论上：每发行 1 个稳定币，银行账户里就有 1 美元储备        │
│                                                             │
└─────────────────────────────────────────────────────────────┘

优点：
✅ 机制简单易懂
✅ 价格最稳定

缺点：
❌ 中心化（公司可以冻结你的账户）
❌ 需要信任发行公司真的有足额储备
❌ 监管风险
```

### 2. 加密资产抵押型（去中心化）

```
代表：DAI

工作原理：
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   你 ────▶ 存入 $150 ETH 抵押品 ────▶ MakerDAO 智能合约     │
│                                               │              │
│                                               ▼              │
│   你 ◀──── 收到 100 DAI（最多） ◀────  铸造 DAI            │
│                                                             │
│   超额抵押：抵押品价值 > 铸造的稳定币价值                     │
│   如果 ETH 跌太多 → 清算机制保护系统                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘

优点：
✅ 去中心化，无人能冻结
✅ 透明，链上可查
✅ 抗审查

缺点：
❌ 资本效率低（150% 抵押率）
❌ 极端行情可能脱锚
❌ 智能合约风险
```

### 3. 算法型（高风险）

```
代表：曾经的 UST（已崩盘）、FRAX

工作原理（简化）：
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   当 1 稳定币 > $1：                                        │
│   系统：铸造更多稳定币 → 增加供给 → 价格下降                 │
│                                                             │
│   当 1 稳定币 < $1：                                        │
│   系统：销毁稳定币 → 减少供给 → 价格上升                     │
│                                                             │
│   通常用另一个代币（如 LUNA）来吸收波动                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘

⚠️ 警告：
2022 年 UST/LUNA 崩盘导致 400 亿美元蒸发
算法稳定币风险极高，新手请远离！
```

## 什么时候用 / 什么时候别用

### ✅ 适合使用稳定币的场景

| 场景 | 原因 |
|------|------|
| 熊市避险 | 把资产换成稳定币，等待时机 |

| 场景 | 原因 |

| 完全信任其安全性 | 脱锚风险始终存在 |

## 它不是什么（常见混淆点）

### ❌ 稳定币不是「绝对稳定」

```
历史脱锚事件：

2022 年 3 月：USDC 短暂脱锚至 $0.87
原因：硅谷银行倒闭，Circle 有 33 亿存在该银行

2022 年 5 月：UST 完全崩盘，从 $1 跌至 $0.01
原因：算法机制失效，死亡螺旋

2023 年：USDT 多次短暂脱锚至 $0.97-0.99
原因：市场恐慌情绪

结论：稳定币 ≠ 100% 安全
```

### ❌ 去中心化不是「完全去中心化」

```
误解：DAI 是完全去中心化的

实际：
• DAI 有大量 USDC 作为抵押品
• 如果 USDC 被冻结，DAI 也会受影响
• MakerDAO 的治理仍然有中心化成分

没有完美的稳定币，只有相对的取舍
```

### ❌ 稳定币不是「存银行一样安全」

```
传统银行存款：
• 有存款保险（FDIC 保 $250,000）
• 有政府背书

稳定币：
• 没有存款保险
• 发行方可能倒闭
• 智能合约可能有 bug
• 可能被黑客攻击

风险级别：稳定币 > 银行存款
```

## 最小例子：理解 DAI 的铸造

### MakerDAO 铸造 DAI 的流程

```
步骤 1：开设金库（Vault）
┌─────────────────────────────────────────┐
│  你在 MakerDAO 开一个 ETH 金库           │
│  类似于在典当行开户                       │
└─────────────────────────────────────────┘

步骤 2：存入抵押品
┌─────────────────────────────────────────┐
│  存入 1 ETH（假设 $2000/ETH）            │
│  抵押品价值 = $2000                      │
└─────────────────────────────────────────┘

步骤 3：铸造 DAI
┌─────────────────────────────────────────┐
│  最低抵押率 150%                         │
│  最多可铸造 = $2000 / 1.5 = 1333 DAI    │
│  安全起见，铸造 1000 DAI                 │
│  实际抵押率 = 2000/1000 = 200%          │
└─────────────────────────────────────────┘

步骤 4：使用 DAI
┌─────────────────────────────────────────┐
│  用 DAI 做任何事：                        │
│  • 交易                                  │
│  • 做 LP                                │
│  • 借贷                                  │
│  • 消费                                  │
└─────────────────────────────────────────┘

步骤 5：还款取回抵押品
┌─────────────────────────────────────────┐
│  还 1000 DAI + 稳定费（利息）            │
│  取回 1 ETH                              │
└─────────────────────────────────────────┘
```

### 计算示例

```python
# 抵押品
eth_amount = 1
eth_price = 2000
collateral_value = eth_amount * eth_price  # $2000

# MakerDAO 参数
min_collateral_ratio = 1.5  # 150%
stability_fee = 0.05  # 5% 年化稳定费

# 最大可铸造
max_dai = collateral_value / min_collateral_ratio  # 1333 DAI

# 安全铸造（200% 抵押率）
safe_dai = collateral_value / 2  # 1000 DAI
current_ratio = collateral_value / safe_dai  # 200%

# 清算价格计算
liquidation_ratio = 1.5
liquidation_price = (safe_dai * liquidation_ratio) / eth_amount
# = (1000 × 1.5) / 1 = $1500/ETH

print(f"抵押品价值: ${collateral_value}")
print(f"铸造 DAI: {safe_dai}")
print(f"当前抵押率: {current_ratio:.0%}")
print(f"清算触发价格: ${liquidation_price}/ETH")
print(f"ETH 需跌 {(eth_price - liquidation_price)/eth_price:.0%} 才会清算")
```

## 新手最常踩的 3 个坑

### 坑 1：认为所有稳定币都一样

```
场景：
你把所有资金放在 USDT
USDT 突然传出储备不足的新闻
恐慌性脱锚到 $0.95
你的资产瞬间缩水 5%

正确做法：
┌─────────────────────────────────────────┐
│  分散持有多种稳定币                       │
│  • 50% USDC                             │
│  • 30% DAI                              │
│  • 20% USDT                             │
│  不要把鸡蛋放在一个篮子里                 │
└─────────────────────────────────────────┘
```

### 坑 2：被算法稳定币的高收益吸引

```
场景：
某算法稳定币承诺 20% APY
你投入 $10,000
一周后项目崩盘，归零

教训：
• 高收益 = 高风险
• 算法稳定币的「稳定」靠的是信心
• 一旦信心崩塌，价格崩盘
• 新手远离算法稳定币！
```

### 坑 3：忽视链上差异

```
场景：
你从交易所提 USDT 到钱包
选了 ERC-20（以太坊）网络
Gas 费花了 $20

其实：
• 同样的 USDT 在不同链上有不同版本
• TRC-20（波场）手续费 $1
• Polygon 手续费几美分
• 但流动性和安全性也不同

正确做法：
• 小额转账选便宜的链
• 大额或长期存储选以太坊主网
• 确认接收方支持你选择的网络！
```

## 自测题

### 基础理解
1. USDT 和 DAI 的本质区别是什么？各自的风险点在哪里？
2. 为什么 DAI 需要超额抵押？如果只抵押 100% 会怎样？
3. 算法稳定币为什么风险高？UST 崩盘的核心原因是什么？

### 实践应用
4. 你有 $10,000 稳定币想长期持有，应该怎么分配？为什么？
5. 你想把稳定币从交易所转到自己钱包，应该考虑哪些因素？

### 深度思考
6. 如果美联储出台严格的稳定币监管政策，对不同类型的稳定币会有什么影响？
7. 为什么说「没有完美的稳定币」？三种类型各自牺牲了什么？

---

## 答案提示

1. USDT 靠银行里的美元支撑，风险在于要信任 Tether 公司；DAI 靠链上加密资产抵押，风险在于抵押品暴跌和智能合约风险
2. 超额抵押是为了应对价格波动；100% 抵押时，价格稍微跌一点就要清算，系统会很脆弱
3. 算法稳定币靠市场信心维持，一旦信心崩塌会形成死亡螺旋；UST 崩盘因为 LUNA 暴跌无法吸收抛压
4. 分散配置：40% USDC + 40% DAI + 20% 其他；避免单一风险
5. 考虑：手续费（不同网络差异大）、到账时间、目标钱包支持的网络、资金安全（大额用主网）
6. 法币抵押型受影响最大（可能被冻结、限制）；加密抵押型相对抗审查；算法型可能被禁止
7. 法币抵押型牺牲了去中心化；加密抵押型牺牲了资本效率；算法型牺牲了稳定性
