# DeepCode 学习框架

> **用 AI 把论文变成代码** — 一份面向新手的完整学习指南

---

## 1. 终极目标

### 解决的问题

你是否遇到过这种情况？

```
看到一篇很厉害的论文 → 想复现代码 → 作者没开源 → 自己写不出来 → 放弃
```

**DeepCode 就是要解决这个问题**：让 AI 自动把论文变成可运行的代码。

### 最终交付物

```
输入：一篇论文 (PDF)
输出：一个完整的代码仓库
      ├── 项目结构
      ├── 核心代码
      ├── 配置文件
      └── 可以跑起来！
```

### 评判标准

- **PaperBench 基准测试**：75.9% 复现分数
- **对比人类专家**：超过 ML 博士生的 72.4%
- **首次超越人类**的论文复现 AI 系统

### 学完能做什么？

| 层次 | 能力 | 应用场景 |
|------|------|----------|
| **理解** | 看懂 DeepCode 的设计思想 | 技术调研、方案评估 |
| **使用** | 用 DeepCode 复现论文 | 科研、学习、项目开发 |
| **改进** | 修改和优化 DeepCode | 二次开发、贡献开源 |

## 4. 单点穿透案例

> **最好的学习方式：跟着一个例子走完全流程**

### 案例入口

[07-单点穿透案例.md](./07-单点穿透案例.md)

### 案例概览

```
案例：实现一个推荐系统论文

输入论文:
  "BPR: Bayesian Personalized Ranking"
  包含: BPR Loss 公式、采样策略、训练流程

DeepCode 执行过程:

  Phase 1 - Blueprint:
    Concept Agent: "这是一个推荐系统论文，核心是 BPR 排序方法"
    Algorithm Agent: "Loss = -∑ log σ(y_ui - y_uj)"
    Planning Agent: → 生成 Blueprint

  Phase 2 - Code Generation:
    生成 config.py      → CodeMem 记录接口
    生成 data_loader.py → CodeMem 更新
    生成 bpr_model.py   → CodeRAG 检索参考实现
    生成 train.py       → 使用 CodeMem 中的接口

  Phase 3 - Verification:
    运行 train.py
    发现: ImportError → 自动修复
    再运行 → 成功!

输出:
  generate_code/
  ├── config.py
  ├── data_loader.py
  ├── bpr_model.py
  ├── train.py
  └── requirements.txt
```
