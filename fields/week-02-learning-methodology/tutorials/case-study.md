# 案例实战：用学习方法体系 7 天入门一个新领域

本案例以技术领域（Kubernetes）为例，展示学习方法的综合运用。如果你是非技术背景，可以跳到文末的"通用案例参考"，那里有学外语、考驾照等更贴近日常的例子。

---

## 环境准备（技术案例必读）

在开始案例之前，如果你想跟着动手操作，需要先准备以下环境：

### 1. 安装 Docker
Kubernetes 需要容器运行环境，首先安装 Docker：
- **Mac**：下载 [Docker Desktop](https://www.docker.com/products/docker-desktop/)，安装后启动
- **Windows**：下载 [Docker Desktop](https://www.docker.com/products/docker-desktop/)，需开启 WSL2
- **Linux**：`sudo apt-get install docker.io`（Ubuntu）或参考官方文档

### 2. 安装 kubectl（K8s 命令行工具）
- **Mac**：`brew install kubectl`
- **Windows**：`choco install kubernetes-cli` 或下载 [官方二进制文件](https://kubernetes.io/docs/tasks/tools/)
- **Linux**：`sudo snap install kubectl --classic`

验证安装：
```bash
kubectl version --client
# 预期输出类似：Client Version: v1.28.0
```

### 3. 安装本地 K8s 集群（推荐 minikube）
- **安装 minikube**：参考 [官方安装指南](https://minikube.sigs.k8s.io/docs/start/)
- **启动集群**：
```bash
minikube start
# 预期输出：
# 😄  minikube v1.32.0 on Darwin
# ✨  Automatically selected the docker driver
# 📌  Using Docker Desktop driver with root privileges
# 🔥  Creating kubernetes in kubernetes cluster...
# 🏄  Done! kubectl is now configured to use "minikube" cluster
```

验证集群运行：
```bash
kubectl cluster-info
# 预期输出：
# Kubernetes control plane is running at https://127.0.0.1:xxxx
```

> **提示**：如果上述准备步骤有困难，也可以仅阅读案例了解学习方法的运用，无需动手操作。

## 案例背景

### Day 2 上午：学习 Pod

**第1步：阅读（15分钟）**
- 看官方文档的 Pod 部分
- 看 2-3 篇中文博客

**第2步：费曼学习法（10分钟）**

小明假装给一个不懂技术的朋友解释：

> "Pod 就像一个小房子。我们的程序（容器）住在这个小房子里。为什么需要这个房子？因为有时候几个程序需要住在一起、共享一些东西（比如文件、网络）。K8s 不直接管理程序，而是管理这些小房子。"

讲的时候发现卡壳：
- ❓ 为什么不直接管理容器？
- ❓ 一个 Pod 里可以放几个容器？

**第3步：补漏洞**
回去查资料，搞懂上面的问题。

**第4步：实践（20分钟）**
```bash
# 创建一个简单的 Pod
kubectl run nginx --image=nginx
# 预期输出：pod/nginx created

# 查看 Pod 状态（等待几秒让 Pod 启动）
kubectl get pods
# 预期输出：
# NAME    READY   STATUS    RESTARTS   AGE
# nginx   1/1     Running   0          30s

# 进入 Pod 看看（输入 exit 退出）
kubectl exec -it nginx -- /bin/bash
# 预期输出：root@nginx:/#（进入了 Pod 内部的命令行）
```

**第5步：记录到闪卡**

```
正面：什么是 Pod？
背面：K8s 最小部署单元，一个或多个容器的组合，共享网络和存储
```

### Day 2-3 的学习内容

| 时间 | 概念 | 大白话解释 |
|-----|------|-----------|
| Day2 上午 | Pod | 装程序的小房子 |

| Day3 上午 | Service | 给一群 Pod 统一门牌号，方便别人找 |
| Day3 下午 | ConfigMap | 配置文件的云端存储 |

**使用的方法**：
- 📝 **费曼学习法**：用大白话解释每个概念
- ⏰ **番茄工作法**：每 25 分钟学一个概念
- 💾 **主动回忆**：做闪卡，合上资料回忆

## Day 4：单点穿透——跑通完整流程

### 目标

部署一个完整的应用，从写 YAML 到能访问。

### 小明的操作

**第1步：写 Deployment YAML**

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-app
        image: nginx
        ports:
        - containerPort: 80
```

**第2步：写 Service YAML**

```yaml
apiVersion: v1
kind: Service
metadata:
  name: my-app-service
spec:
  selector:
    app: my-app
  ports:
  - port: 80
    targetPort: 80
  type: NodePort
```

**第3步：部署 + 验证**

```bash
# 部署（确保 deployment.yaml 和 service.yaml 在当前目录）
kubectl apply -f deployment.yaml
# 预期输出：deployment.apps/my-app created

kubectl apply -f service.yaml
# 预期输出：service/my-app-service created

# 验证（等待约30秒让 Pod 启动完成）
kubectl get pods
# 预期输出：
# NAME                      READY   STATUS    RESTARTS   AGE
# my-app-5d7b8c9f6-abc12    1/1     Running   0          45s
# my-app-5d7b8c9f6-def34    1/1     Running   0          45s
# my-app-5d7b8c9f6-ghi56    1/1     Running   0          45s

kubectl get services
# 预期输出：
# NAME             TYPE       CLUSTER-IP      PORT(S)        AGE
# my-app-service   NodePort   10.96.xxx.xxx   80:30080/TCP   1m

# 在 minikube 环境中访问服务
minikube service my-app-service --url
# 会输出一个 URL，在浏览器打开即可看到 nginx 欢迎页面
```

### 遇到问题怎么办

小明遇到了一个问题：Pod 启动失败，状态是 ImagePullBackOff。

**排查流程**：
```bash
# 看详细信息
kubectl describe pod my-app-xxx

# 发现：镜像拉取失败

# 解决：换成国内镜像源
image: nginx:latest → image: docker.io/library/nginx:latest
```

**关键学习**：遇到问题 → describe 看详情 → 根据错误信息解决

**使用的方法**：
- 🎯 **刻意练习**：不只是看，必须自己动手做
- 🔄 **单点穿透**：用一个例子打通整个流程

## Day 6：实战项目

### 任务

把公司的一个真实服务部署到 K8s。

### 过程

1. **分析需求**：这个服务需要什么？
   - 容器镜像：已有
   - 配置文件：需要用 ConfigMap
   - 环境变量：需要用 Secret
   - 对外访问：需要 Service

2. **写配置文件**：参考之前学的，边写边查

3. **部署测试**：
   - 第一次失败 → 查日志 → 修改配置
   - 第二次失败 → 发现端口写错 → 修改
   - 第三次成功！

4. **文档记录**：把过程写成文档，以后可以复用

**使用的方法**：
- 💪 **刻意练习**：真实项目，有挑战
- 🧠 **元认知**：遇到问题分析原因，而不是瞎试
