# 案例实战：用学习方法体系 7 天入门一个新领域

## 案例背景

小明是一名后端工程师，公司新项目需要用到 Kubernetes（K8s），但他完全不懂。领导说"给你一周时间，学会基本操作"。

小明决定用我们学过的学习方法体系来攻克这个挑战。

让我们跟着小明，看他如何一步步学习。

## Day 1：终局定向 + 建立框架

### 早上：问自己 WHY（为什么学）

小明先问自己三个问题：

```
┌─────────────────────────────────────────────────────────┐
│                   WHY - 终局定向                         │
└─────────────────────────────────────────────────────────┘

1. K8s 解决什么问题？
   ──▶ 管理大量容器，自动部署、扩缩容、故障恢复

2. 学完我要能做什么？
   ──▶ 能把公司的服务部署到 K8s 集群，能处理基本运维问题

3. 谁来评判我学得怎么样？
   ──▶ 能让同事的服务跑起来就算合格
```

**关键动作**：明确学习目标，不是"学 K8s"，而是"能部署服务+基本运维"。

```
                        │
                        │           ┌─ 查看日志
```

### Day 2 上午：学习 Pod

**第1步：阅读（15分钟）**
- 看官方文档的 Pod 部分
- 看 2-3 篇中文博客

**第2步：费曼学习法（10分钟）**

小明假装给一个不懂技术的朋友解释：

> "Pod 就像一个小房子。我们的程序（容器）住在这个小房子里。为什么需要这个房子？因为有时候几个程序需要住在一起、共享一些东西（比如文件、网络）。K8s 不直接管理程序，而是管理这些小房子。"

讲的时候发现卡壳：
- ❓ 为什么不直接管理容器？
- ❓ 一个 Pod 里可以放几个容器？

**第3步：补漏洞**
回去查资料，搞懂上面的问题。

**第4步：实践（20分钟）**
```bash
# 创建一个简单的 Pod
kubectl run nginx --image=nginx

# 查看 Pod 状态
kubectl get pods

# 进入 Pod 看看
kubectl exec -it nginx -- /bin/bash
```

**第5步：记录到闪卡**

```
正面：什么是 Pod？
背面：K8s 最小部署单元，一个或多个容器的组合，共享网络和存储
```

### Day 2-3 的学习内容

| 时间 | 概念 | 大白话解释 |
|-----|------|-----------|
| Day2 上午 | Pod | 装程序的小房子 |
| Day2 下午 | Deployment | 管理一群 Pod 的管家，保证数量达标 |
| Day3 上午 | Service | 给一群 Pod 统一门牌号，方便别人找 |
| Day3 下午 | ConfigMap | 配置文件的云端存储 |

**使用的方法**：
- 📝 **费曼学习法**：用大白话解释每个概念
- ⏰ **番茄工作法**：每 25 分钟学一个概念
- 💾 **主动回忆**：做闪卡，合上资料回忆

## Day 4：单点穿透——跑通完整流程

### 目标

部署一个完整的应用，从写 YAML 到能访问。

### 小明的操作

**第1步：写 Deployment YAML**

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-app
        image: nginx
        ports:
        - containerPort: 80
```

**第2步：写 Service YAML**

```yaml
apiVersion: v1
kind: Service
metadata:
  name: my-app-service
spec:
  selector:
    app: my-app
  ports:
  - port: 80
    targetPort: 80
  type: NodePort
```

**第3步：部署 + 验证**

```bash
# 部署
kubectl apply -f deployment.yaml
kubectl apply -f service.yaml

# 验证
kubectl get pods      # 看到 3 个 pod running
kubectl get services  # 看到 service 的 NodePort
curl localhost:30080  # 能访问到 nginx 页面！
```

### 遇到问题怎么办

小明遇到了一个问题：Pod 启动失败，状态是 ImagePullBackOff。

**排查流程**：
```bash
# 看详细信息
kubectl describe pod my-app-xxx

# 发现：镜像拉取失败

# 解决：换成国内镜像源
image: nginx:latest → image: docker.io/library/nginx:latest
```

**关键学习**：遇到问题 → describe 看详情 → 根据错误信息解决

**使用的方法**：
- 🎯 **刻意练习**：不只是看，必须自己动手做
- 🔄 **单点穿透**：用一个例子打通整个流程

## Day 6：实战项目

### 任务

把公司的一个真实服务部署到 K8s。

### 过程

1. **分析需求**：这个服务需要什么？
   - 容器镜像：已有
   - 配置文件：需要用 ConfigMap
   - 环境变量：需要用 Secret
   - 对外访问：需要 Service

2. **写配置文件**：参考之前学的，边写边查

3. **部署测试**：
   - 第一次失败 → 查日志 → 修改配置
   - 第二次失败 → 发现端口写错 → 修改
   - 第三次成功！

4. **文档记录**：把过程写成文档，以后可以复用

**使用的方法**：
- 💪 **刻意练习**：真实项目，有挑战
- 🧠 **元认知**：遇到问题分析原因，而不是瞎试
